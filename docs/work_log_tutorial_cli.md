# Tutorial CLI 통합 작업 기록

## 단계별 진행 내역
- **문서 검토**: `tutorial_implementation_plan.md`, `tutorial_system_full.md`, `user_scenario_onboarding.md`의 단계 흐름, 성공/실패 처리, UI 요구사항을 재확인했습니다.
- **런타임/루프 분기 정리**: `runtime`, `loop`, `lifecycle`, `navigation`에 튜토리얼 플래그와 스톰/사이클 제어 분기를 추가하고, STOP 판정 실패·사망 시 `retryStep()`을 타도록 수정했습니다.
- **튜토리얼 매니저 구현**: `TutorialManager`에서 단계 상태 관리, 이벤트 트리거(사이클 시작/실드 스폰/스톰 활성화), 성공 조건 체크, 재시도/완료 흐름, 진행도 저장(`tutorialProgress`, `tutorialCompleted`)을 구현했습니다.
- **UI/환경 적용**: 튜토리얼 전용 오버레이 활성화/비활성, 단계별 HUD 가시성, 힌트/재시도 메시지, 하이라이트 처리를 정비하고, 로비/타이틀에서 튜토리얼 진입 경로를 연결했습니다.
- **저장 및 재진입**: 기본 세이브 스키마에 튜토리얼 필드를 포함하고, 완료 후 로비 전환 및 재연습 시 1단계부터 시작하도록 했습니다.

## 현재 상태
- 코드 변경을 통해 튜토리얼 4단계 흐름(기본 조작 → RUN/WARNING/STOP → 위험 인지 → 실전)과 분기 로직을 반영했습니다.
- 자동 테스트 스크립트는 제공되지 않아 별도 실행하지 않았습니다.
