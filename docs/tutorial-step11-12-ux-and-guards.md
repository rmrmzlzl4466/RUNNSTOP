# Tutorial Expansion – Step 11 & 12

## 작업 요약
- 튜토리얼 진입/종료 페이드 효과와 입력 차단 클래스를 도입해 UX를 부드럽게 만들었습니다.
- 튜토리얼 재진입/중복 실행을 방지하고, 런타임 존재 여부를 확인하는 가드 로직을 추가했습니다.
- 페이지 새로고침/중단 시 상태를 기록해 로드 후 로비로 안전 복귀하도록 했습니다.

## 세부 변경 내역
1. **페이드 및 입력 차단**
   - `tutorialUI`에 `fadeIn/fadeOut`을 추가해 단계 표시와 완료 연출에 적용.
   - `tutorial.css`에 `tutorial-lock`, `fade-in/out` 스타일과 pointer-events 제어를 추가해 튜토리얼 중 다른 입력을 막고 버튼 중복 클릭을 방지.

2. **재진입/중복 실행 가드**
   - `TutorialManager`에 `isRunning` 플래그를 도입해 중복 시작을 막고, 완료 시 해제.
   - `checkStepCondition`, `onStepComplete`, `retryStep`에 런타임 유효성 검사를 추가.

3. **새로고침 방어 및 복귀 흐름**
   - `main.js`에서 튜토리얼 실행 중 `beforeunload` 시 세션 스토리지에 플래그를 남기고, 페이지 로드 후 로비로 복귀.
   - 튜토리얼 진입/완료 시 `tutorial-lock` 클래스를 토글하여 UX 안정화.

## 메모
- 튜토리얼 전용 오버레이는 페이드 전환을 통해 화면 깜빡임 없이 자연스러운 흐름을 제공합니다.
- 튜토리얼이 완료되면 런타임을 일반 모드로 재구성하고 로비로 복귀한 뒤 사용자가 직접 게임을 시작하도록 유지합니다.
