# 네온 글리치 보완 기획 (1-2) (2025-12-31 21:58)

## 범위
- 1) 팔레트 리디자인
- 2) 타일 표면 질감/글로우 추가

## 1) 팔레트 리디자인

### 목표
- UI 톤(네온/사이버/글리치)과 스테이지 비주얼 간 괴리 해소
- 초반부터 컨셉 각인: 기본 스테이지도 네온 베이스로 시작

### 문제 인식
- DEFAULT 팔레트가 범용 컬러에 가깝고, 메인 컨셉(네온/글리치)에 비해 밋밋함
- 초반 스테이지가 보수적 색감이라 UI 대비 톤 일치감이 떨어짐

### 기획 제안
- DEFAULT 팔레트를 폐기하거나 NEON_BASE로 대체
- Stage 1~2를 NEON_BASE 계열로 통일 (기본 톤 고정)
- 팔레트 구성은 다음 룰로 설계
  - 배경 톤: 딥 그린/블랙 계열
  - 강조 톤: 네온 그린, 시안, 마젠타
  - 안전색: 네온 그린 고정
  - 위험색: 마젠타/레드 계열

### 예시 팔레트 (초안)
- NEON_BASE
  - colors: #00ff41, #00d4ff, #ff0040, #8b00ff, #11ff9a, #0d2b1f
  - safeColor: #00ff41
  - dangerColor: #ff0040

### 적용 방안
- 파일: `assets/js/data/stages.js`
  - `STAGE_PALETTES.DEFAULT` -> `NEON_BASE`로 교체하거나 DEFAULT 자체를 네온 계열로 변경
  - Stage 1, 2의 theme paletteId를 `NEON_BASE`로 설정
- 파일: `assets/js/data/themes.js`
  - UI와 연동되는 테마 팔레트가 있다면 동기화

### 검증 포인트
- 타이틀/로비 UI와 스테이지 색감의 톤 매칭 체감
- 튜토리얼/Stage 1의 첫 인상에서 네온 글리치 컨셉이 즉시 느껴지는지

## 2) 타일 표면 질감/글로우 추가

### 목표
- 타일이 단순 평면 색상이 아니라, 사이버틱한 재질감을 느끼게 하기
- STOP 구간에서 안전 타일이 더 뚜렷하게 보이도록 UX 개선

### 문제 인식
- 타일의 시각 정보가 색상에만 의존
- 글리치/스캔라인/네온 같은 메인 컨셉을 타일에서 체감하기 어려움

### 기획 제안
1) 스캔라인/노이즈 오버레이
   - 타일 위에 얇은 스캔라인 또는 미세 노이즈를 얹어 전자 디스플레이 느낌 강화
2) 안전 타일 글로우
   - STOP 상태에서 safeColor 타일에만 약한 글로우/밝기 강조
   - 안전/위험 피드백이 더 명확해짐
3) 글로우는 과하지 않게 제한
   - 시각적 피로 최소화
   - 저사양 기기 대비로 토글 가능하게 설계

### 적용 방안
- 파일: `assets/js/core/renderer.js`
  - 타일 렌더링 루프에서 안전색 타일에 가벼운 오버레이/글로우 처리
  - 스캔라인/노이즈는 전역 오버레이 레이어로 추가 가능
- 파일: `assets/css/style.css`
  - 캔버스 위에 얇은 scanline overlay 레이어 추가 가능
  - `#game-surface::after` 같은 형태로 구현 가능

### 구현 아이디어 (렌더링 기준)
- 타일 그리기 후, safeColor 타일에만 `ctx.globalAlpha`로 약한 발광 블렌드
- 전역 스캔라인은 낮은 opacity로 캔버스 위에 고정

### 검증 포인트
- STOP 구간에서 안전 타일이 명확히 눈에 띄는지
- 전체 화면이 과하게 어두워지거나 눈이 피로하지 않은지
- 모바일 성능 저하 없이 유지되는지
